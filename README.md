
Sentiment-Analysis 情感分析项目

一、项目简介

本项目基于 OCEMOTION 口语情感数据集搭建中文二分类情感分析系统，整体技术栈：jieba分词 + TF-IDF特征提取 + 逻辑回归（LogisticRegression）

- 数据集总量 3.5万条 口语文本，按照 训练集:验证集:测试集 = 8:1:1  分层划分
- 原生数据集存在正负样本不均衡问题（正面样本仅占 28%），通过 class_weight='balanced' 类别权重均衡优化，正面样本召回率由 43% 提升至 66%，模型均衡性大幅改善
- 基准模型测试集整体准确率可达 81.55%
- 配套可视化评估图表、文本预测记录持久化存储、实时交互式预测功能
- 系统扩展对比 6 种不平衡处理策略（类加权、SMOTE、阈值移动、特征选择、组合优化），验证各方案优劣

 四、数据集说明

OCEMOTION 数据集：口语化中文情感语料，原生多标签分类

| 项目 | 说明 |
|:---|:---|
| 原始标签 | 7 类细粒度情感：happiness、love、surprise、sadness、anger、fear、disgust |
| 正面映射 | happiness、love、surprise → 标签 1 |
| 负面映射 | 其余 4 类情绪（sadness、anger、fear、disgust）→ 标签 0 |
| 样本总量 | 35,279 条有效样本（清洗去空后）|
| 样本分布 | 负面 25,500 条（72%），正面 9,779 条（28%），正负比约 2.61:1 |

数据预处理流程：
1. 清洗非中文字符、统一小写
2. jieba 精确模式分词，空格分隔分词结果
3. 剔除空文本脏数据，保证训练数据有效性


五、核心模型流程
1. 分层数据集拆分（防止随机划分分布偏移）
- 第一步：总数据 80% 训练集 + 20% 临时集
- 第二步：临时集对半拆分，各 10% 作为验证集、测试集
- 全程使用 `stratify` 分层抽样，保证三集合正负样本比例和原数据集一致
- 
划分结果：训练集 28,223 条 / 验证集 3,528 条 / 测试集 3,528 条

2. 特征工程
- 仅用训练集拟合 TF-IDF 词袋词典（`max_features=10000`）
- 验证/测试集只做特征转换，彻底杜绝数据泄露

3. 模型训练与调优
- 基础模型：逻辑回归 `LogisticRegression(max_iter=1000)`
- 关键优化：`class_weight='balanced'` 自动加权平衡正负样本损失

 4. 六组对照实验设计

| 编号 | 模型方案 | 核心策略 |
|:---:|:---|:---|
| ① | 基线模型 | 无任何不平衡处理 |
| ② | 类别加权 | `class_weight='balanced'` |
| ③ | Borderline-SMOTE | 合成少数类过采样 |
| ④ | 阈值移动 | 验证集搜索最优阈值（0.39）|
| ⑤ | 特征选择 | 卡方+互信息联合筛选 Top3000 |
| ⑥ | 组合优化 | 特征选择 + SMOTE |

5. 多维度评估体系
- 量化指标：训练/验证/测试集三路准确率、精确率、召回率、F1分数、AUC
- 可视化指标：准确率对比图、混淆矩阵热力图、ROC曲线、样本分布统计图

七、模型效果指标

各模型核心指标对比
| 模型方案 | 测试 Acc | 加权 F1 | AUC | 正面 Recall |
|:---|:---:|:---:|:---:|:---:|
| ① 基线无优化 | 81.35% | 0.79 | 0.825 | **43%** |
| ② class_weight 类加权 | 78.60% | 0.79 | **0.830** | **66%** |
| ③ Borderline-SMOTE | 74.83% | 0.76 | 0.808 | **67%** |
| ④ **阈值移动（最优）** | **81.41%** | **0.81** | 0.825 | 54% |
| ⑤ 特征选择 | 81.52% | 0.79 | 0.824 | 43% |
| ⑥ 特征选择+SMOTE组合 | 73.41% | 0.74 | 0.810 | 69% |

 关键结论
- 综合均衡最优：阈值移动模型（加权 F1=0.81，全场最高），无需重新训练，部署成本最低
- 少数类识别最优：class_weight（正面召回 66%）与 SMOTE（67%），其中 class_weight 不引入噪声，工程落地首选
- 基线缺陷明显：未做任何平衡时，正面样本召回率仅 43%，大量正面文本被漏判
- 策略不可盲目叠加：特征选择 + SMOTE 组合出现性能衰减，证明多策略叠加可能引入双重噪声


八、自定义修改指南
1. 修改结果保存路径
修改代码顶部 `SAVE_DIR` 变量：
python
# 示例：改到E盘文件夹

2. 修改数据集划分比例
调整两次 `train_test_split` 的 `test_size` 参数即可自定义训练/验证/测试占比

3. 更换分类标签规则
修改 `map_label()` 函数内 `positive_labels` 列表，自定义哪些情绪归为正向

4. 适配 Linux/Mac 系统
- 文件路径：将 `\` 替换为 `/`
- 中文字体：将 `SimHei` 替换为系统中文字体（如 `WenQuanYi Zen Hei`）
